# ぷよぷよ — Aseprite Edition

自作の `puyopuyo.aseprite` のドット絵をそのまま使ったぷよぷよです。
ゲーム内のぷよは全部このファイルの絵から描画されます。

## 遊び方

**index.html をブラウザで開くだけ**で遊べます（ダブルクリックでOK）。

### モード

| モード | 始め方 | 内容 |
|---|---|---|
| とことん | ENTER | ひたすら連鎖を組むソロモード |
| おじゃまサバイバル | S | 時間経過でおじゃまが降ってくる。レベルが上がるほど激しくなるので、連鎖で相殺しながら生き残ろう |
| COM対戦 | C | AIの相手とおじゃまを送り合うオフライン対戦。ツモ順はおたがい同じで公平 |
| オンライン対戦 | V | 友だちとネット越しに対戦 |

| キー | 操作 |
|---|---|
| ← → | 移動 |
| Z | 左回転 |
| X / ↑ | 右回転 |
| ↓ | 高速落下 |
| Space | 一気に落とす |
| P | ポーズ（ソロのみ） |
| M | サウンド ON/OFF |
| R | リスタート / 対戦後の再戦 |
| Q | タイトルにもどる（対戦中は降参） |

同じ色を4つ以上つなげると消えます。連鎖でスコアがどんどん増えます（ぷよぷよ通と同じ計算式）。

## オンライン対戦のやり方

1. ふたりとも同じゲームページを開く
2. ひとりがタイトル画面で **V** → **部屋を作る** → 合言葉（`puyo-xxxx`）が表示される
3. 合言葉を相手に伝える（LINEなど何でもOK）
4. 相手は **V** → **部屋に入る** → 合言葉を入力 → **つなぐ**
5. 3カウントで同時スタート！ふたりとも同じツモ順で公平です

### 対戦ルール

- 連鎖すると相手に**おじゃまぷよ**（白ぷよ）を送れます（スコア70点ごとに1個）
- 自分に降ってくる予定のおじゃまは盤面の左上に「×N」で予告表示
- 予告されたおじゃまは、自分も連鎖すれば**相殺**して打ち消せます
- 全消しするとおじゃま30個ぶんの大攻撃！
- 対戦後: **R** = もう一回（両方が押すと再戦） / **Q** = タイトルへ

通信は WebRTC（PeerJS）でブラウザ同士が直接つながるため、サーバー代はかかりません。
ただし部屋の仲介にインターネット接続が必要です。

### 友だちと遊ぶには（ページの公開）

**GitHub Pages が無料で簡単:**

1. GitHubのアカウントを作り、新しいリポジトリを作る
2. このフォルダのファイルを全部アップロード
3. リポジトリの Settings → Pages → Branch を `main` にして保存
4. 発行されたURL（`https://ユーザー名.github.io/リポジトリ名/`）を友だちに送る

同じWi-Fi内なら公開しなくてもOK: 片方のPCでこのフォルダで
`python -m http.server 8642` を実行し、もう片方が
`http://<そのPCのIPアドレス>:8642` を開けば遊べます。

## 絵を描き変えたとき（再エクスポートの手順）

`puyopuyo.aseprite` を編集して保存したら:

1. **`export.bat` をダブルクリック**
2. ブラウザをリロード（F5）

これだけで新しい絵がゲームに反映されます。

`export.bat` は中で次のことをしています:

```bat
aseprite.exe -b --split-layers "puyopuyo.aseprite" ^
  --filename-format "{layer}_{frame}" ^
  --sheet "assets\puyos.png" ^
  --data "assets\puyos.json" ^
  --sheet-type packed
```

- `--split-layers` … レイヤーごとにバラして書き出す（**入力ファイル名より前に書く**のが重要）
- `--filename-format "{layer}_{frame}"` … 各コマに `red_0` のような名前を付ける（ゲームはこの名前で色とフレームを判別）
- `--sheet` / `--data` … スプライトシートPNGと座標JSONを出力
- 最後にJSONを `assets/puyos.js` に変換（`index.html` をダブルクリックで開いたときでも読み込めるようにするため）

> Asepriteの場所が変わったら `export.bat` の先頭の `ASEPRITE=` のパスを直してください。
> 現在は Steam 版 (`C:\Program Files (x86)\Steam\steamapps\common\Aseprite\aseprite.exe`) を指しています。

## .aseprite ファイルのルール

ゲームは書き出したデータを起動時に自動で読むので、座標を手で直す必要はありません。
ただし次の構造は守ってください:

- **1レイヤー = 1色のぷよ**。レイヤー名がそのまま色の名前になります
  （現在: `red` / `blue` / `green` / `orange` / `while` ※`while` は `white` として扱われます）
- **フレームの意味**（各レイヤー共通）:
  - フレーム1つ目 … 通常の顔
  - フレーム2つ目 … まばたき・着地した瞬間の顔
  - フレーム3つ目 … 消えるときの顔
- **キャンバスサイズは 16×16** のまま
- フレームを増やしても壊れません（余分は使われず、足りない分は近いフレームで代用されます）
- **レイヤーを追加すると新しい色を増やせます**。ゲームで使う色数は
  `index.html` 内の `const COLOR_COUNT = 4;` を変えてください（5にすると白ぷよも降ってきます）
- **おじゃまぷよの絵は白ぷよ（while レイヤー）を使っています**。専用の絵にしたい場合は、
  `ojama` という名前のレイヤーを追加すれば自動でそちらが使われます

## ファイル構成

```
puyopuyo/
├── puyopuyo.aseprite   ← 素材の原本（これを編集する）
├── export.bat          ← 再エクスポート用（ダブルクリック）
├── index.html          ← ゲーム本体
├── README.md
└── assets/             ← export.bat が自動生成（手で編集しない）
    ├── puyos.png       … スプライトシート
    ├── puyos.json      … 各コマの座標データ
    └── puyos.js        … puyos.json のJS版（file://で開いた時用）
```
